# Citrail ― 引用を辿る

1本の学術論文（DOIまたはPMID）を起点に、その論文が引用している「過去文献」と、
その論文を引用している「未来文献」をネットワーク図として表示する論文読解支援ツールです。

## 使い方（初回セットアップ）

1. https://openalex.org で無料アカウントを作成する
2. https://openalex.org/settings/api でAPIキーをコピーする
3. `index.html` をブラウザで開く（ダブルクリックでOK）
4. 右上の「APIキー設定」からキーを貼り付けて保存する
5. DOIまたはPMIDを入力して「ネットワークを作成」

APIキーはブラウザ内（Local Storage）にのみ保存され、OpenAlex以外へは送信されません。
無料枠は毎日リセットされ、個人利用には十分です。

## ファイル構成と役割

- `index.html` — 画面構造
- `style.css` — デザイン
- `app.js` — 画面全体の制御（起動・フィルタ・共有/PNG・入力〜ネットワーク作成）
- `panels.js` — 詳細パネル・展開・お気に入り・上位リスト
- `ui.js` — モーダル・APIキー・ヘルプ・起動時の自動作成
- `api.js` — OpenAlexからの論文データ取得と入力の正規化
- `graph.js` — ネットワーク図の生成と操作（Cytoscape.js）
- `vendor/cytoscape-3.28.1.min.js` — Cytoscape.jsの本体（CDNではなく同梱。改ざんリスクの回避のため）
- `storage.js` — お気に入り・APIキーのブラウザ内保存
- `config.example.js` — 設定ファイルの例（config.jsにコピーして使用、任意）
- `docs/` — 仕様書などの文書置き場

## 開発メモ

- 研究種別（RCT／メタ解析・SR／レビュー／観察研究／ガイドライン）は、PMIDがある論文は
  PubMedの公式Publication Typeで判定し、それ以外はタイトルからの暫定判定にフォールバック
- 一般公開は「利用者が自分の無料APIキーを登録する方式（仕様書23.3の案A）」を採用。
  中継API（案B）は「キー登録すら不要にしたい」場合のフェーズ2以降の課題とする
- 重要度基準の切り替えは、仕様書14.2の4基準（被引用数・ハブ性・古い順・新しい順）を
  「上位リスト」パネル（トップ10表示＋ノードへジャンプ）として実装
- 動作確認済みの版にはGitHub上でタグを付ける
