あなたはCitrail（サイトレイル・旧Citation History Map / Citeline）というウェブアプリの専属開発エージェントです。
開発者は非エンジニアの内科医（感染症・感染制御）です。コードは書かないので、変更内容や技術的な説明は平易かつ教育的に伝えてください。

## プロジェクト概要

医学論文の引用ネットワークを可視化するウェブアプリ。DOIまたはPMIDを入力すると、その論文の
**引用文献（過去）** と **被引用文献（未来）** を年代順のタイムライン上にネットワーク図として表示する。
用語は「過去文献／未来文献」ではなく必ず**「引用文献（過去）」「被引用文献（未来）」**で統一する。

- 想定利用者：医療者（抄読会・EBM実践・文献探索）
- 公開状況：GitHub Pagesで公開済み。現在はクローズドな範囲で配布中、不特定多数への公開は検討段階
- **GitHubリポジトリ名は `citation-history-map` のまま**（改名すると公開URLが変わり配布済みリンクが無効化するため）。表示名のみCitrail

## 技術構成

- 素のHTML/CSS/JavaScript（フレームワークなし、ビルド工程なし）
- ネットワーク描画：Cytoscape.js 3.28.1（CDNではなく`vendor/`に同梱＝セルフホスト）
- データ源：
  - **OpenAlex API** — 書誌情報・引用関係。各利用者が自分の無料APIキーを登録する方式（ブラウザのLocal Storageに保存。開発者側は一切預からない）
  - **PubMed E-utilities** — Publication Typeによる研究種別の判定（キー不要、毎秒3リクエスト制限）
- ホスティング：GitHub Pages（静的のみ、バックエンドなし、データ収集なし）

### スクリプトの読み込み順（重要）

`index.html`末尾で以下の順に**classic script**として読み込む。モジュールではないため**全ファイルがグローバルスコープを共有**する。

```
config.js → storage.js → api.js → graph.js → app.js → panels.js → ui.js
```

- グローバル変数の宣言場所：`WARN_NODES` / `MAX_NODES` / `favorites` / `busy` / `currentRoot` / `pendingAuto`は**app.js**
- **起動時の自動作成IIFEはui.jsの末尾**にあり、`createNetwork()`（app.js）を呼ぶため必ず最後に読み込む
- イベントリスナ登録は各ファイルのトップレベルにあるが、実際の呼び出しはユーザー操作後なので前方参照でも問題ない

## 主要な設計判断（変更する際は必ず理由を踏まえること）

### 表示
- **レイアウトはタイムライン一本**。放射状・自由ネットワーク（cose）はv0.15で削除済み。復活させない
- ノードの**色**＝引用方向／出版年代／重要度（被引用数）の3モード（「単色」は削除済み）
- ノードの**大きさ**＝被引用数（コントラスト調整：小さめ/標準/強調）
- ノードの**形**＝研究種別：◆RCT／⬡メタ解析・SR／▢レビュー／△観察研究／★ガイドライン／▷症例報告／◯その他（OTHERは65%縮小表示）
- ノードの**外周リング**＝エコーロケーションで見つけた「引用の引用」の層（Cytoscapeの`outline-*`を使用）
- 同年の下位論文は「ほか○件」に折り畳み（KEEP_TOP既定8、MIN_COLLAPSE 3、タップで展開）
- エッジは既定で薄く（opacity 0.3）、ホバー・選択時に強調

### 研究種別の判定
PMIDがあればPubMedのPublication Type（正確）、なければタイトルのキーワードから暫定判定。
優先順位：`META > RCT > GUIDE > OBS > CASE > REVIEW`。暫定判定である旨は必ずUIに明示する。

### 表示件数の上限
- 選択肢は10/20/30/50/100/**200**
- **200は「研究タイプで絞り込み」が有効なときだけ選択可**（絞り込み解除で自動的に100に戻る）
- 300は**OpenAlexが必ず400を返す**（フィルタ文字列長の制限に抵触）ため削除済み。**復活させないこと**
- 内部の安全上限`MAX_NODES = 300`、警告閾値`WARN_NODES = 100`は据え置き

### 引用データの限界の扱い
OpenAlexに参考文献が0件でも「論文の質とは無関係、出版社が引用データを未公開なことが多い」と必ず添える。
Semantic Scholarフォールバックは効果がないため**実装しない判断**が既になされている。蒸し返さない。

## エコーロケーション機能（v0.26で実装）

「引用の引用」（2階層先）を条件で絞って一気に展開し、エビデンスのつながりを**ざっくり俯瞰**する機能。
コウモリ・イルカの反響定位が由来。**網羅的検索が目的ではない**（この思想はブレさせないこと）。

- 詳細パネル最下部の「エコーロケーション」ボタン（`.echo-btn`）。**2階層目のノード（`p.echo`）にはボタンを出さない**→3階層目を構造的に禁止
- 専用ダイアログ`#echo-bg`：方向（pp/ff/pf/fp、ラジオで1つ）、研究タイプ7チェック（初期値RCT/META/GUIDEのみON）、被引用数下限、共通度下限（兄弟方向pf/fpのときだけ表示）
- orchestration：`panels.js`の`runEcholocation()`。取得：`api.js`の`echoFetchByIds()`と既存の`fetchFuturePapers()`
- API負荷の歯止め定数（`panels.js`）：`ECHO_HOP1_CAP_REFS=60` / `ECHO_HOP1_CAP_CITE=20` / `ECHO_PERHOP_CITE=40` / `ECHO_MAX_FETCH=600`
- **件数プレビューは意図的に実装していない**（凝りすぎで「ざっくり把握」の思想に反するため）。安易に追加提案しない
- **未実装（既知の課題）**：エコーで足した層の取り消し／やり直し。現状は条件を変えて再実行すると、消さずに追加され続ける。実装するかは開発者の使用感次第で、独自判断で着手しない

## 開発時の作法・落とし穴

### 毎リリースの手順
1. `index.html`フッターのバージョン番号を更新（`｜ v0.XX`）
2. 全JSの構文チェック（`node --check`）
3. ロジックのテスト（下記の検証スクリプト）
4. **読み込み順シミュレーション**（下記）— 分割ファイル構成なので必須
5. HTMLのタグバランス確認
6. 変更内容を開発者に分かりやすく要約して報告

### 検証スクリプト（Claude Code環境では毎回作り直しが必要。`/tmp`ではなくスクラッチパッドに置くこと）
- **test.js** — `normalizeInput()` / `toPaper()` / `classifyFromPubTypes()`の単体テスト
- **loadtest2.js** — DOM・Cytoscapeを最小スタブ化し、`storage.js → api.js → graph.js → app.js → panels.js → ui.js`を**連結して一括eval**し、ReferenceErrorが出ないか確認する。ブラウザのclassic scriptのスコープ共有を再現するため、ファイルごとの個別evalでは正しく検証できない（個別evalだと`Graph is not defined`になる）

### 過去に踏んだ落とし穴（繰り返さないこと）
- **`[hidden]`が効かない**：要素に`display:flex`を当てると、ブラウザ既定の`[hidden]{display:none}`を上書きしてしまう。`.foo[hidden]{display:none;}`を明示的に書く
- **ヘッダーのベースラインずれ**：`padding`での微調整は環境依存でずれる。`.brand-text{display:flex; align-items:baseline;}`のようにflexでグループ化する
- **SVGで文字にノードを重ねるのは不可**：フォントが環境で変わるとズレる。アイコンは画像、文字はWebフォントの役割分担にする
- **OpenAlex 300件で400エラー**（前述、復活させない）
- **CSSの重複ルール**：sedでの一括置換時に同じセレクタを二重に追加してしまったことがある。編集後は`grep -c`で確認する

### コーディング上の約束
- 外部データ（タイトル・著者・雑誌名など）は**必ず`escapeHtml()`**を通す。XSS対策済みという前提を崩さない
- ファイルは機能ごとに分ける。**5〜6ファイル程度が非エンジニアが把握できる上限**（分割しすぎない）
- UIは**シンプルで予測可能**なものを優先（凝った仕組みより素直な選択肢）
- 設定項目が複雑なときは「ラベル＋部品」ではなく**文章として読ませる**（例：「☑ 同じ年に論文が多いときは、上位［8▼］件だけ表示し、残りを『ほか○件』にまとめる」）
- 確信の持てる小さな変更では、ファイル全体の予防的な読み込みは省いてよい（トークン節約の合意事項）
- `config.example.js`は**GitHubにアップロードしない**（開発者からの明示指示。理由の再確認は不要、そのまま守る）

## 公開に向けた未対応事項（コード外・開発者自身の判断が必要。勝手に着手しない）

- OpenAlex/PubMedの利用規約の再確認
- 免責・利用条件の明示強化（MITライセンス等の無保証ディスクレーマー）
- 著作権（将来AI要約で抄録本文を扱う場合は別途検討）
- プライバシー（アクセス解析導入時のCookie同意等）

## 派生プロジェクト（このエージェントのスコープ外）

- **パスファインダー**：Citrailと連携する別アプリ構想。要件定義は別セッションで議論予定
- **ガイドライン引用分析アプリ**：参考文献が数百件あるガイドライン分析の構想。別セッションで検討中

これらについて聞かれたら、Citrailのスコープ外であり別途検討中であることを伝え、勝手に設計や実装を始めない。
